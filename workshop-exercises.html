Why Codex Here

  • Trace issues across frontend, backend, and tests in one loop.
  • Generate constrained diffs with minimal blast radius.
  • Close the loop with test verification plus browser and visual QA.

Client Outcomes

  • Exercises 1-2: codify repeatable setup and contract-drift debugging.
  • Exercises 3-4: run incident-style triage and stabilize KPI/API contracts.
  • Exercises 5-10: ship features, automate QA, and expand product readiness.

Exercise 1: Install Codex (App or CLI) and Connect Project

10-20 minutes Onboarding

Goal: get every participant running Codex against the local repository.

Choose your path

  1. Open Codex App and sign in with ChatGPT.
  2. Select this repository folder.
  3. Run these orientation prompts:
    • Summarize this repository structure.
    • List the intentionally failing tests and what each one checks.
    • Map dashboard summary data from backend to frontend.
  1. Install CLI: 
    npm install -g @openai/codex
# or
brew install --cask codex
  2. Start Codex inside repo: 
    codex
  3. Sign in and run the same orientation prompts as App path.

Acceptance criteria

  • Codex (App or CLI) can read local files accurately.
  • You can run prompts and get architecture-accurate answers.

Exercise 3: Incident Response - Low Stock KPI Mismatch

10-20 minutes Incident Response

Goal: triage and resolve a production-style KPI mismatch affecting planning decisions.

Codex setup (App or CLI)

  • Codex App: Reuse your repo session in Codex App and run: Treat this as incident response: validate low-stock KPI mismatch evidence, apply minimal fix for reorder-point equality, rerun regressions, and summarize RCA.
  • Codex CLI: Run codex in repo root (or reuse your open session), then run the same prompt.

Incident brief

  • Planning report understated low-stock exposure for procurement review.
  • Open incident packet: docs/incidents/INC-2026-03-low-stock-kpi.md.
  • Working hypothesis: equality at reorder point is excluded from KPI count.

Run regression checks

source .venv/bin/activate
cd tests
pytest backend/test_bug_regressions.py -q

Tasks

  1. Confirm symptom and impacted KPI from incident packet and tests.
  2. Use Codex to locate low-stock calculation in backend summary endpoint.
  3. Patch the conditional to include equality case.
  4. Re-run the regression file.
  5. Add a short RCA note to output/incident-rca.md (3-5 bullets).

Acceptance criteria

  • Items at reorder point are counted as low stock.
  • Backend regression for this KPI incident passes.
  • output/incident-rca.md documents root cause and mitigation.
Mark exercise complete
